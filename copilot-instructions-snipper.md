## Maintainability workflow

For each task that plans, writes, modifies, refactors, or reviews production code, invoke the `/maintainable-code` skill before implementation.

Continue to apply the skill after each coherent implementation slice and before completion. Do not treat the skill as only a final review.

If you delegate production-code changes, instruct each delegated agent to apply the `/maintainable-code` skill throughout its task.

Prefer repository-configured quality tools. Do not install an analyzer or change repository configuration unless the task includes that work or the user approves it.
